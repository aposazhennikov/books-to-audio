# Ответы на вопросы о Review

## ❓ Вопрос 1: После разбиения на главы как я должен проверять? Последовательно каждую главу?

### Ответ: НЕТ, не по главам отдельно!

Система проверяет **всю книгу целиком** за один проход. Ты просматриваешь проблемы **последовательно** (одну за другой), но они могут быть из **любой главы**.

### Пример работы:

```bash
python -m book_normalizer.cli process books/mybook.txt --interactive
```

**Система показывает**:
```
Issue 1 of 12  ← Всего 12 проблем во всей книге
Chapter: "Глава 1"  ← Эта проблема в главе 1
Type: punctuation
...

Issue 2 of 12
Chapter: "Глава 5"  ← Следующая проблема в главе 5
Type: ocr_artifact
...

Issue 3 of 12
Chapter: "Глава 2"  ← Теперь проблема в главе 2
Type: punctuation
...
```

**Ты НЕ проверяешь**:
- ❌ Сначала всю главу 1
- ❌ Потом всю главу 2
- ❌ И т.д.

**Ты проверяешь**:
- ✅ Проблему 1 (может быть в любой главе)
- ✅ Проблему 2 (может быть в любой главе)
- ✅ Проблему 3 (может быть в любой главе)
- ✅ И так далее по всем проблемам

### Можно пропускать

Если не хочешь сейчас решать какую-то проблему, выбираешь `[3] Skip`, и система перейдёт к следующей. К пропущенным можно вернуться позже.

## ❓ Вопрос 2: Будет ли результат моих выборов сохраняться автоматически и использоваться в будущем?

### Ответ: ДА! Абсолютно автоматически!

### Как это работает:

#### 1️⃣ Первая книга (ты принимаешь решения)

```bash
python -m book_normalizer.cli process book1.txt --interactive

# Проблема: ,, → ,
# Ты: [1] Accept
# Система: ✅ Запомнила!

# Проблема: 0 → о (в слове "м0сква")
# Ты: [1] Accept
# Система: ✅ Запомнила!
```

**Файлы памяти обновились**:
- `data/user_memory/punctuation_memory.json` ← сохранено `,,` → `,`
- `data/user_memory/correction_memory.json` ← сохранено `0` → `о`

#### 2️⃣ Вторая книга (автоматически!)

```bash
python -m book_normalizer.cli process book2.txt
# Даже БЕЗ --interactive!

# Вывод:
# Review session created: 10 total, 3 auto-resolved, 7 pending.
#                                   ^^^^^^^^^^^^^^^^^^^
#                      3 проблемы исправлены АВТОМАТИЧЕСКИ!
```

Если в `book2.txt` встретятся:
- `,,` → автоматически исправится на `,`
- `0` в слове → автоматически исправится на `о`

**Тебе НЕ НУЖНО заново принимать решения!**

#### 3️⃣ Третья, четвёртая, N-ная книга

Каждая новая книга будет использовать ВСЮ накопленную память. Чем больше книг ты обработаешь, тем больше проблем будет исправляться автоматически!

### Статистика памяти

```bash
# После обработки 10 книг
Review session created: 15 total, 12 auto-resolved, 3 pending.
                                   ^^^ 80% автоматически!

# После обработки 50 книг
Review session created: 20 total, 18 auto-resolved, 2 pending.
                                   ^^^ 90% автоматически!
```

## 🎯 Практический сценарий

### День 1: Обрабатываешь первую книгу

```bash
python -m book_normalizer.cli process books/book1.txt --interactive

# Находишь 20 проблем
# Принимаешь 20 решений (Accept/Reject/Edit)
# Время: ~10 минут
```

**Система запомнила**: 20 типов исправлений

### День 2: Обрабатываешь вторую книгу

```bash
python -m book_normalizer.cli process books/book2.txt --interactive

# Находит 25 проблем
# 15 исправлено автоматически (уже знакомые)
# Нужно решить: 10 новых проблем
# Время: ~5 минут
```

**Система запомнила**: +10 типов = 30 всего

### День 3: Обрабатываешь третью книгу

```bash
python -m book_normalizer.cli process books/book3.txt --interactive

# Находит 30 проблем
# 25 исправлено автоматически
# Нужно решить: 5 новых проблем
# Время: ~2 минуты
```

**Система запомнила**: +5 типов = 35 всего

### Месяц спустя: Обрабатываешь 50-ю книгу

```bash
python -m book_normalizer.cli process books/book50.txt
# Даже без --interactive!

# Находит 40 проблем
# 38 исправлено автоматически
# Нужно решить: 2 новые проблемы
# Время: ~1 минута
```

**95% проблем исправляется АВТОМАТИЧЕСКИ!**

## 📝 Краткое резюме

### Вопрос 1: Как проверять?
✅ **Всю книгу за один проход**, проблемы показываются последовательно (не по главам)

### Вопрос 2: Сохраняются ли решения?
✅ **ДА, автоматически!** Каждое решение запоминается и применяется ко всем будущим книгам

### Бонус: Экономия времени
✅ Первая книга: ~10 минут
✅ Десятая книга: ~2 минуты
✅ Пятидесятая книга: ~30 секунд

**Система становится умнее с каждой книгой!**
