## Что НЕ обнаруживается (ограничения)

Текущая система **не обнаруживает**:

1. **Настоящие орфографические ошибки** (требуется словарь):
   - "превет" вместо "привет"
   - "координатр" вместо "координатор"
   - "интелект" вместо "интеллект"

2. **Отсутствие пунктуации**:
   - Предложение без точки в конце
   - Отсутствие запятой перед союзом

3. **Грамматические ошибки**:
   - "в течении времени" vs "в течение времени"
   - "прийти" vs "придти"

4. **Семантические ошибки**:
   - "компания" (организация) vs "кампания" (мероприятие)

## Как использовать

### 1. Просмотр обнаруженных проблем

```bash
# Обработать книгу
python -m book_normalizer.cli process books/mybook.txt -v

# Посмотреть JSON с проблемами
cat data/user_memory/review_sessions/session_*.json | jq .pending_issues
```

### 2. Интерактивное исправление

```bash
# Запустить интерактивный режим
python -m book_normalizer.cli process books/mybook.txt --interactive

# Для каждой проблемы выбрать действие:
# [1] Accept  - Принять исправление
# [2] Reject  - Отклонить
# [3] Skip    - Пропустить
# [4] Edit    - Ручное редактирование
# [5] Exit    - Выйти
```

### 3. Автоматическое применение памяти

После того, как ты один раз принял исправление в интерактивном режиме, система запомнит его и **автоматически применит** при следующей обработке любой другой книги!

```bash
# Первый раз - интерактивно принимаешь исправление "0" → "о"
python -m book_normalizer.cli process book1.txt --interactive

# Второй раз - автоматически применится для book2.txt
python -m book_normalizer.cli process book2.txt
# Вывод: "Review session created: 10 total, 5 auto-resolved, 5 pending."
```

## Файлы памяти

Система хранит твои решения в:

- `data/user_memory/punctuation_memory.json` - решения по пунктуации
- `data/user_memory/correction_memory.json` - решения по орфографии/OCR

Формат записи:

```json
{
  "entries": [
    {
      "original": ",,",
      "replacement": ",",
      "confirmed": true,
      "count": 5,
      "last_seen": "2026-03-11T22:19:08Z"
    }
  ]
}
```
