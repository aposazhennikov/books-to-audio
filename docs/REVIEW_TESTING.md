# Тестирование функций Review (проверка орфографии и пунктуации)

## Что проверяется

### 1. Проверка пунктуации (`PunctuationIssueDetector`)

Система ищет следующие проблемы:

#### Высокая серьёзность (high):
- **Повторяющиеся запятые**: `,,` → `,`
  - Пример: "Еще один пример,, с двойной запятой." → "Еще один пример, с двойной запятой."

#### Средняя серьёзность (medium):
- **Отсутствие пробела после пунктуации**: `слово,слово` → `слово, слово`
  - Пример: "Привет,мир" → "Привет, мир"
- **Два точки подряд** (вероятно, сломанное многоточие): `..` → `…`
  - Пример: "Текст.. продолжение" → "Текст… продолжение"

#### Низкая серьёзность (low):
- **Пробел перед пунктуацией**: `слово ,` → `слово,`
  - Пример: "Привет , мир" → "Привет, мир"
- **Чрезмерное количество восклицательных/вопросительных знаков**: `!!!` (3+)
  - Пример: "Ого!!!!" → "Ого!!"

### 2. Проверка орфографии и OCR-артефактов (`OcrSpellingDetector`)

#### Высокая серьёзность (high):
- **Смешанная кириллица и латиница**: `привeт` (где `e` — латинская)
  - Автоматически транслитерируется в кириллицу
- **Цифры вместо букв внутри слов**:
  - `0` вместо `О`: "м0сква" → "москва"
  - `3` вместо `З`: "ска3ал" → "сказал"
  - `6` вместо `б`: "6ыл" → "был"

#### Средняя серьёзность (medium):
- **Подозрительно длинные последовательности**:
  - 5+ согласных подряд: "встрчстврнстй"
  - 5+ гласных подряд: "аааааа"

## Как проверить

### Вариант 1: Без интерактива (просто посмотреть проблемы)

```bash
# Обработать книгу
python -m book_normalizer.cli process books/test_review.txt --out output/test_review -v

# Посмотреть найденные проблемы в JSON
cat data/user_memory/review_sessions/session_*.json
```

### Вариант 2: С интерактивом (исправить проблемы)

```bash
# Запустить с интерактивным режимом
python -m book_normalizer.cli process books/test_review.txt --out output/test_review --interactive

# Система покажет каждую проблему и спросит, что делать:
# [1] Accept    - Принять предложенное исправление
# [2] Reject    - Отклонить (оставить как есть)
# [3] Skip      - Пропустить (решить позже)
# [4] Edit      - Ручное редактирование
# [5] Exit      - Выйти из режима проверки
```

### Вариант 3: Отключить определённые проверки

```bash
# Отключить проверку пунктуации
python -m book_normalizer.cli process books/test.txt --skip-punctuation

# Отключить проверку орфографии
python -m book_normalizer.cli process books/test.txt --skip-spellcheck

# Отключить обе
python -m book_normalizer.cli process books/test.txt --skip-punctuation --skip-spellcheck
```

## Память системы

Система **запоминает** твои решения и автоматически применяет их в будущем:

1. **Память пунктуации** (`data/user_memory/punctuation_memory.json`):
   - Если ты принял исправление `,,` → `,`, оно применится автоматически при следующей обработке

2. **Память исправлений** (`data/user_memory/correction_memory.json`):
   - Если ты принял исправление `превет` → `привет`, оно применится автоматически

## Тестовый файл

Файл `books/test_review.txt` содержит следующие ошибки для проверки:

1. ✅ **Двойная запятая**: "Еще один пример,, с двойной запятой." → обнаружена!
2. ❌ **Орфография "превет"**: не обнаружена (требуется словарь)
3. ❌ **Отсутствие точки в конце**: не обнаружена (сложно определить)
4. ❌ **Лишние пробелы**: не обнаружены (нормализация удаляет их автоматически)
5. ❌ **Неправильный пробел после точки**: требует дополнительного паттерна

## Как улучшить детекторы

Текущие детекторы используют **регулярные выражения** и **эвристики**, но не проверяют:
- Реальную орфографию слов (нужен словарь русского языка)
- Грамматические ошибки (нужен парсер)
- Контекстные ошибки (нужна языковая модель)

Для более продвинутой проверки можно интегрировать:
- **Yandex.Speller API** или **LanguageTool** для орфографии
- **pyaspeller** для проверки русского языка
- **rupo** для пунктуации
