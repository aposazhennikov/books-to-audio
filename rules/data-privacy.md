# Data Privacy

- Do not commit real books, generated audio, user manifests, private paths, tokens, or logs containing personal data.
- Use synthetic data for examples, tests, documentation, fixtures, and screenshots.
- Treat books, extracted chunks, normalized text, previews, prompts, audio, logs, user paths, and manifests as private by default.
- Before adding diagnostics, reports, support bundles, or logs, check whether they can leak user content.
- Do not include book excerpts or private runtime paths in reports unless the user explicitly requested that disclosure.
